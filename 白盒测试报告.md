## 发现的 BUG 列表

### 🔴 严重 BUG

#### BUG #1: checkDisconnectedPlayerTurn 缺少 await ✅ 已修复
- **文件**: `server/src/gameRoom.ts:306`
- **问题**: `this.saveState()` 没有 await，可能导致状态未持久化
- **影响**: 断线重连状态可能丢失
- **修复**: 函数改为 async，添加 await，更新所有调用处

---

#### BUG #2: handlePlayHand 潜在无限循环 ✅ 已修复
- **文件**: `server/src/gameRoom.ts:628-633`
- **问题**: 当玩家出完最后一张牌时，如果其他所有玩家都已出完牌（handCount === 0），while 循环会无限执行
- **修复**: 在计算 nextIndex 前先检查是否只剩一人有牌，满足则触发 GAME_END 并 return

---

#### BUG #3: Durable Object 休眠后 sessions 丢失 ✅ 已修复
- **文件**: `server/src/gameRoom.ts:256-285`
- **问题**: `sessions` 是内存 Map，Durable Object 休眠后丢失，且未使用 `serializeAttachment` 持久化 WebSocket 绑定的 playerId
- **修复**: 
  - `handleJoinRoom` 添加 `ws.serializeAttachment({ playerId })`
  - 新增 `restoreSessions()` 方法，在构造函数调用

---

### 🟡 中等 BUG

#### BUG #4: 断线重连后不自动发送 JOIN_ROOM ✅ 已修复
- **文件**: `src/stores/gameStore.ts`
- **问题**: wsManager 重连成功后触发 onOpen，但 gameStore 未注册 onOpen 回调重新发送 JOIN_ROOM
- **修复**: 在 `initialize` 中注册 `wsManager.onOpen` 回调，重连后发送 JOIN_ROOM

---

#### BUG #5: 后端缺少 CHAT_MESSAGE 处理 ✅ 已修复
- **文件**: `server/src/gameRoom.ts:359-386`
- **问题**: `handleMessage` 的 switch 语句没有处理 `CHAT_MESSAGE` 类型
- **修复**: 添加 `case MessageType.CHAT_MESSAGE` 和 `handleChatMessage` 方法

---

#### BUG #6: 前端 PLAY_HAND 处理未更新自己的 myHand ✅ 已修复
- **文件**: `src/stores/gameStore.ts:369-405`
- **问题**: 移除乐观更新后，自己出牌时前端不更新 myHand，完全依赖服务器 STATE_SYNC
- **修复**: PLAY_HAND 处理中，如果是自己出的牌，从 myHand 移除对应牌

---

#### BUG #7: 前端 shuffleDeck 实现与后端不一致 ✅ 已修复
- **文件**: `src/utils/dealer.ts:52-67`
- **问题**: 前端 shuffleDeck 使用临时变量和条件判断，后端使用解构赋值，代码不一致
- **修复**: 统一使用解构赋值写法 `[shuffled[i], shuffled[j]] = [shuffled[j]!, shuffled[i]!];`
